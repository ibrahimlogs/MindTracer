# Devpost Description

## Inspiration

Two students can both answer “10” and still need very different help. MindTrace began from that gap.

## What it does

MindTrace compares learner reasoning, forms misconception hypotheses, verifies the support need, gives the smallest useful intervention, and checks transfer on a new context.

## How we built it

We used Next.js App Router, TypeScript, Tailwind CSS, shadcn/ui-style primitives, Prisma, PostgreSQL-compatible storage, Zod contracts, OpenAI SDK integration boundaries, Vitest, and Playwright.

## Challenges we ran into

The hardest part was keeping AI flexible without letting it become an unconstrained tutor. We solved this with curated IDs, schemas, verification templates, safety checks, and deterministic fallbacks.

## Accomplishments we are proud of

The judge demo tells the full product story quickly while staying honest about cached fallback, deterministic evaluation, and live-service status.

## What we learned

Learning support gets safer when the system treats diagnosis as a hypothesis to verify, not a label to announce.

## What is next for MindTrace

Configure live services, record the demo video, deploy, publish the repository, and submit.
